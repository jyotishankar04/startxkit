The `doctor` command runs a diagnostic check on your StartXKit project to ensure everything is configured correctly. Use it to catch misconfigurations early.

## Usage

```sh
npx @startxkit/cli doctor
```

## Checks Performed

- **Config file** ‚Äî verifies `startxkit.config.json` exists and is valid JSON
- **Package file** ‚Äî verifies `package.json` exists
- **Module directory** ‚Äî confirms the configured module directory exists
- **Framework** ‚Äî validates the framework is one of `express`, `fastify`, or `hono`
- **Architecture** ‚Äî validates the architecture is `minimal`, `modular`, or `layered`
- **Package manager** ‚Äî validates it is `pnpm`, `npm`, `yarn`, or `bun`

## Example Output

```text
‚úì startxkit.config.json found
‚úì package.json found
‚úì Module directory "src/modules" exists
‚úì Framework: express
‚úì Architecture: modular
‚úì Package manager: pnpm

All checks passed!
```

## Troubleshooting

If any check fails, you'll see a `‚úó` marker with a description:

```text
‚úì startxkit.config.json found
‚úó package.json not found ‚Äî run "npm init" first
‚úì Module directory "src/modules" exists
‚úó Unknown framework "koa" in config
```

Fix the reported issues and run `npx @startxkit/cli doctor` again.
