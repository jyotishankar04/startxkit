# 008 - Docs Site and User Guides

Status: Done

## Goal

Provide a docs site that explains installation, usage, supported stacks, and the roadmap.

## What is in place

- Next.js docs app under `apps/docs`.
- MDX content structure for:
  - getting started
  - frameworks
  - architectures
  - CLI commands
  - roadmap
  - changelog
- Docs navigation and shared layout components.

## Important files

- `apps/docs/app`
- `apps/docs/components`
- `apps/docs/content/docs`
- `apps/docs/lib`
- `README.md`
- `CONTRIBUTING.md`

## Behavior implemented

- Documents supported frameworks: Express, Fastify, and Hono.
- Documents supported architectures: minimal, modular, and layered.
- Documents the commands exposed by the CLI.
- Documents current limitations and future roadmap items.

## Current outcome

The repository already includes a public-facing documentation site that matches the implemented feature set.
